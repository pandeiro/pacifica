# Technical Reference: Observability

## Overview

Pacifica exposes structured logs, Prometheus metrics, and PostgreSQL metrics for Grafana/Loki/Prometheus monitoring. The API auto-instruments all endpoints. The scraper service writes an audit trail to the `scrape_logs` table on every execution.

## Endpoints

| Endpoint | Port | Purpose |
|----------|------|---------|
| `/api/health` | 4900 | Readiness probe — checks DB connectivity |
| `/api/health/scrapers` | 4900 | Per-scraper status, staleness, consecutive failures |
| `/metrics` | 4900 | Prometheus metrics (HTTP request rate, latency, errors) |
| `:9187/metrics` | 9187 | PostgreSQL exporter (connections, cache hit, disk) |

### `/api/health`

```json
{
  "status": "healthy",
  "service": "pacifica-api",
  "version": "dev",
  "database": "connected"
}
```

Returns `healthy` only if a `SELECT 1` against PostgreSQL succeeds. Use as Docker/NGINX healthcheck target.

### `/api/health/scrapers`

Returns per-scraper health with staleness detection:

```json
{
  "status": "healthy",
  "scrapers": [
    {
      "name": "noaa_tides",
      "last_run_at": "2026-03-21T17:54:44Z",
      "last_status": "success",
      "last_duration_ms": 27800,
      "last_records_created": 528,
      "is_stale": false,
      "minutes_since_last_success": 4.4,
      "consecutive_failures": 0
    }
  ],
  "total_scrapers": 9,
  "healthy_count": 9,
  "stale_count": 0,
  "failed_count": 0
}
```

- **Stale**: no successful run within 2x the scraper's expected interval
- **Critical**: no successful run within 3x expected interval, or 3+ consecutive failures

Overall `status` is `healthy` (0 stale, 0 failed), `degraded` (1-3 stale), or `critical` (>3 stale or any consecutive failures).

## Data Sources

### Prometheus Metrics (`/metrics`)

Auto-generated by `prometheus-fastapi-instrumentator`. Key metrics:

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `http_requests_total` | counter | method, handler, status | Total requests by route |
| `http_request_duration_seconds` | histogram | method, handler | Request latency |
| `http_requests_in_progress` | gauge | method, handler | Active requests |
| `http_request_size_bytes` | summary | handler | Incoming request size |
| `http_response_size_bytes` | summary | handler | Outgoing response size |

Excluded from instrumentation: `/api/health`, `/api/version`, `/metrics`.

### PostgreSQL Exporter (`:9187`)

`prometheuscommunity/postgres-exporter` sidecar connected to the same database. Key metrics:

| Metric | Description |
|--------|-------------|
| `pg_up` | 1 if connected, 0 if not |
| `pg_stat_activity_count` | Active connections by state |
| `pg_stat_database_tup_fetched` | Rows fetched |
| `pg_stat_database_tup_inserted` | Rows inserted |
| `pg_stat_bgwriter_buffers_checkpoint` | Checkpoint writes |

### Loki (Structured Logs)

Both API and scraper containers output to stdout via Docker's JSON log driver. To send JSON to Loki, set `LOG_FORMAT=json` in the environment. With `LOG_FORMAT=pretty` (default for dev), output is human-readable colored console format.

**API logs** (structlog):
```json
{"event": "version_requested", "level": "info", "timestamp": "2026-03-21T17:55:00Z", "logger": "api"}
```

**Scraper logs** (structlog):
```json
{"event": "scrape_completed", "scraper": "noaa_tides", "records": 528, "duration_ms": 27800, "level": "info", "timestamp": "2026-03-21T17:55:00Z"}
```

Structured log events from scrapers:

| Event | When | Key Fields |
|-------|------|------------|
| `scrape_started` | Scraper begins | `scraper` |
| `scrape_completed` | Scraper succeeds | `scraper`, `records`, `duration_ms` |
| `scrape_failed` | Scraper raises exception | `scraper`, `error`, `duration_ms` |
| `scraper_discovered` | Scheduler finds a scraper class | `scraper`, `schedule` |
| `scraper_running` | Scheduler invokes a scraper | `scraper` |
| `scraper_completed` | Scheduler run finishes | `scraper`, `records` |
| `scraper_failed` | Scheduler run fails | `scraper`, `error`, `retry` |

### `scrape_logs` Table

Every scraper execution writes a row. Queryable directly from Grafana's PostgreSQL data source.

```sql
SELECT scraper_name, started_at, finished_at,
       EXTRACT(EPOCH FROM (finished_at - started_at)) * 1000 AS duration_ms,
       status, records_created, records_updated, records_skipped,
       error_message
FROM scrape_logs
ORDER BY started_at DESC
LIMIT 50;
```

Columns:

| Column | Type | Description |
|--------|------|-------------|
| `scraper_name` | text | Class name: `noaa_tides`, `noaa_water_temp`, `south_coast_divers`, `acs_la`, `inaturalist`, `daveyslocker`, `dana_wharf`, `harbor_breeze`, `island_packers` |
| `started_at` | timestamptz | Run start time |
| `finished_at` | timestamptz | Run end time |
| `status` | text | `success`, `failure`, or `partial` |
| `records_created` | int | New rows inserted |
| `records_updated` | int | Existing rows updated |
| `records_skipped` | int | Duplicates/out-of-range skipped |
| `error_message` | text | Exception text (only on failure) |

Index: `idx_scrape_logs_scraper_time ON scrape_logs (scraper_name, started_at DESC)`

## Docker Services

| Service | Image | Health | Resource Limits |
|---------|-------|--------|-----------------|
| `postgres` | `timescale/timescaledb:latest-pg16` | `pg_isready` | 512M mem, 1 CPU |
| `api` | `ghcr.io/pandeiro/pacifica-api` | `curl /api/health` | 256M mem, 1 CPU |
| `scraper` | `ghcr.io/pandeiro/pacifica-scraper` | none (restart: always) | 1G mem, 2 CPU |
| `postgres-exporter` | `prometheuscommunity/postgres-exporter:v0.15.0` | none | 64M mem, 0.25 CPU |

The scraper has higher CPU/memory limits because Playwright (Harbor Breeze) launches a headless Chromium.

## Recommended Alert Rules

| Alert | Condition | Severity |
|-------|-----------|----------|
| Scraper stale | No success within 2x expected interval | warning |
| Scraper critically stale | No success within 3x expected interval | critical |
| Scraper consecutive failures | 3+ failures in a row | critical |
| API 5xx spike | >5 5xx in 5 min | critical |
| API latency | p95 > 2s | warning |
| NOAA API down | 0 successful NOAA fetches in 12h | critical |
| Ollama fallback | LLM fallback rate > 0 in 12h | warning |
| DB connection pool | >80% pool utilization for >5 min | warning |
| Disk usage | PostgreSQL volume > 80% | warning |
